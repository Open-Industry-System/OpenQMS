## 1. 项目亮点

### 一句话概括

**以"知识图谱 + AI 智能推荐"为核心的新一代质量管理 SaaS 平台，让制造业质量管理从"流程流转"升级为"知识驱动"。**

### 核心亮点速览

| 维度 | 亮点 |
|------|------|
| 技术创新 | 国内首个将**图数据库（Neo4j）+ LLM + RAG**应用于质量管理全流程的产品，FMEA 知识图谱变更影响分析从人工数天缩短至秒级 |
| 市场定位 | 切入 **¥500 亿+** 的中国智能制造质量软件市场，现有竞品全部缺失 AI 和知识图谱能力，属于**蓝海窗口期** |
| 商业模式 | SaaS 订阅 + 私有部署双模式，产品线维度计费，天然具备**数据网络效应**——客户越多推荐越准，形成竞争壁垒 |
| 合规壁垒 | 深度覆盖 ISO 9001:2015 和 IATF 16949:2016 共 18 个关键条款，汽车行业认证审核的**刚需工具** |
| 团队优势 | 技术团队具备全栈开发 + 图数据库 + AI/ML 能力，产品设计已完成详细的 PRD 和技术架构设计 |
| 政策红利 | 智能制造、工业软件国产替代、新能源汽车质量合规三重政策叠加 |

### 项目当前阶段

已完成：产品需求文档（PRD v1.2，1973 行）、技术架构设计（含 17 个模块规格、数据模型、API 设计）、核心状态机原型验证、项目脚手架搭建。

## 2. 项目概述

### 2.1 我们要解决什么问题

制造业企业的质量管理普遍面临三大痛点：

| 痛点 | 现状 | 后果 |
|------|------|------|
| **经验随人走** | FMEA 和 8D 报告散落在 Excel 和个人电脑中，工程师离职则知识流失 | 重复踩坑，同一失效模式在不同产品线反复出现 |
| **审核费时费力** | IATF 16949 审核需要准备大量证据链，人工翻阅数月的历史文档 | 审核准备周期长达 2-3 个月，一次不通过损失巨大 |
| **变更追溯靠人脑** | 一个工序参数变更影响哪些 FMEA 和控制计划？全靠工程师记忆和逐条排查 | 遗漏可能导致批量质量事故和召回风险 |

### 2.2 我们的解决方案

OpenQMS 是一个以**产品线(Product Line)**为数据组织维度的智能质量管理平台，核心架构为：

```
前端（React 18+）→ API 网关 → 微服务（FastAPI）
    ├── 核心业务层：FMEA / 8D-CAPA / SPC / 供应商质量 / 客户质量 / 控制计划
    ├── 智能服务层：AI 推荐引擎 / 语义搜索 / RAG 服务 / 变更影响分析
    └── 数据层：PostgreSQL + Neo4j 知识图谱 + Elasticsearch + Milvus 向量库
```

**三条核心数据链串联全部模块：**

1. **产品实现链**：APQP → DFMEA → PFMEA → 控制计划 → SPC → MSA → PPAP
2. **问题解决链**：SPC 异常 / IQC 拒收 / 客诉 → 8D/CAPA → 根因分析 → 关联 FMEA → 经验入库
3. **治理保障链**：质量目标 → 内部审核 → 管理评审 → 仪表盘 KPI

### 2.3 目标客户

| 优先级 | 客户画像 | 核心需求 | 市场规模（国内） |
|--------|---------|---------|----------------|
| ★★★ | 汽车零部件 Tier 1/2 供应商 | IATF 16949 认证合规，FMEA/SPC/PPAP/8D 完整闭环 | ~5,000 家 |
| ★★ | 电子制造企业（消费/工业） | SPC 监控、来料检验、客诉管理 | ~8,000 家 |
| ★ | 医疗器械制造企业 | ISO 13485 合规、CAPA 和风险管理 | ~3,000 家 |

## 4. 产品与技术创新

### 4.1 技术创新点

#### 创新一：FMEA 知识图谱（国内 QMS 首创）

传统 QMS 将 FMEA 存储为二维表单，无法表达失效模式之间的复杂关联。OpenQMS 采用 Neo4j 图数据库原生存储 FMEA 的**13 种节点类型**和**17 种关系类型**，实现：

- **变更影响秒级追溯**：修改一个设计参数，图遍历瞬间定位所有受影响的失效模式和关联控制计划
- **风险热力图可视化**：力导向图按 RPN 值映射节点颜色深浅，风险分布一目了然
- **跨产品线知识复用**：从产品线 A 的 PFMEA 自动发现与产品线 B 相似的失效模式

#### 创新二：AI 智能推荐引擎（三层递进）

| 层级 | 能力 | 技术实现 |
|------|------|---------|
| 第一层（检索式） | 语义搜索 + 历史相似推荐 | Elasticsearch + Milvus 向量检索 + RAG |
| 第二层（生成式） | FMEA 草稿生成 + 8D 报告草拟 | LLM（私有部署 Qwen/ChatGLM）+ 人工审核 |
| 第三层（主动式） | 风险预警推送 + 质量目标推荐 | 图神经网络 + 时序预测 |

**安全边界**：所有 AI 输出强制标注"AI 建议"标签，须经质量工程师审核方可发布，推荐附带相似度分数和数据来源。

#### 创新三：PostgreSQL ↔ Neo4j 双库协同架构

| 场景 | 数据源 | 优势 |
|------|--------|------|
| FMEA 编辑器（表格模式） | PostgreSQL | ACID 事务保证，结构化查询 |
| 知识图谱可视化 | Neo4j | 图遍历查询，O(1) 复杂度多跳关系 |
| 变更影响分析 | Neo4j | Cypher 图算法，秒级追溯全部影响链 |
| AI 语义搜索 | Elasticsearch + Milvus | 全文检索 + 向量相似度 |

### 4.2 产品功能矩阵（17 个模块）

| 模块 | 核心能力 | 差异化 |
|------|---------|--------|
| DFMEA 编辑器 | 系统→子系统→零部件三级展开，功能树分析，设计参数矩阵，接口失效分析 | 知识图谱驱动，与 DVP 验证联动 |
| PFMEA 编辑器 | 工序流编辑，控制计划一键生成，特性矩阵表 | AI 推荐历史失效模式，与控制计划双向联动 |
| 知识图谱可视化 | 力导向图，风险热力图，双击展开子图，路径高亮 | 13 节点类型 + 17 关系类型的完整建模 |
| 8D/CAPA 工作流 | D1-D8 八步问题解决，状态机流转，超时自动升级 | D4/D5 阶段 AI 推荐根因和措施 |
| SPC 控制图 | 7 种图表类型，8 大判异准则自动检测 | 异常触发→自动关联 FMEA→一键启动 CAPA |
| 控制计划 | PFMEA 联动生成，特殊特性清单，SPC 监控关联 | AI 校验 PFMEA→CP 一致性 |
| 供应商质量 | IQC 来料检验，SCAR 管理，供应商审核与绩效评价 | AQL 抽样方案自动计算，IQC 不良→FMEA 关联 |
| 客户质量 | 客诉管理，RMA 退货分析，客户 PPM 看板 | 客诉→8D→FMEA 全链路追溯 |
| 全局知识库 | 跨产品线 FMEA 经验聚合，失效模式搜索引擎 | 数据脱敏自动沉淀，新产品线研发助手 |
| 内部审核 | 三方审核管理，不符合项→CAPA 联动 | 风险导向审核检查表 |
| 管理评审 | 13 项输入数据自动汇总，评审措施跟踪 | 一键生成管理评审报告 |
| 特殊特性管理 | CC/SC 贯穿标识，符号转换表 | 各模块自动同步标注 |
| MSA 测量系统分析 | GR&R/偏倚/线性/稳定性分析 | 与控制计划测量设备自动关联 |
| APQP 项目策划 | 五阶段门管理，五大工具串联，甘特图 | 阶段门 AI 完整性检查 |
| 质量目标管理 | 三级目标树（公司→产品线→过程） | AI 基线推荐 |
| AI 助手 | 语义搜索、FMEA 推荐、8D 推荐、影响分析 | 私有部署 LLM，数据不外传 |
| 仪表盘 | 角色化 KPI 看板，产品线健康度热力图 | 多维度下钻，自定义布局 |

## 10. 风险分析与缓解

| # | 风险 | 等级 | 缓解策略 |
|---|------|------|---------|
| R1 | **LLM 幻觉风险**：AI 推荐错误信息可能导致质量决策失误 | 🔴 极高 | 严格"AI 建议→人工审核"双节点；优先推荐已验证历史数据；私有部署可控模型 |
| R2 | **用户采纳阻力**：质量工程师习惯 Excel，抵触新系统 | 🟠 高 | Excel 一键导入导出；UI 对标 Excel 操作习惯；提供行业 FMEA 模板降低启动成本 |
| R3 | **知识图谱性能**：大规模图谱查询响应慢 | 🟠 高 | Phase 1-2 用 PostgreSQL JSONB 过渡；图缓存层；单次查询深度 ≤3 层 |
| R4 | **数据隐私**：客户质量数据通过 LLM API 外泄 | 🟡 中 | 私有部署 LLM；全局知识库脱敏；敏感字段不入 RAG 索引；180 天审计日志 |
| R5 | **竞品跟进**：Siemens 等 12-18 个月内跟进 AI | 🟡 中 | 数据飞轮先发优势 + 汽车零部件细分市场深度 + 核心算法专利申请 |
| R6 | **Neo4j-PostgreSQL 同步一致性问题** | 🟠 高 | PostgreSQL 为写主库；Kafka CDC 异步同步（延迟 ≤5s）；每日全量校验自动修复 |
| R7 | **标准合规性缺失** | 🟠 高 | 按合规性对照表逐条覆盖（18 个 IATF 条款）；邀请 IATF 审核员参与需求评审 |
