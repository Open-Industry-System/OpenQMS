# FMEA 图谱轻量 UI 优化设计

日期：2026-06-18

## 背景

当前 FMEA 图谱页面在中文界面中仍直接显示英文节点/边枚举，例如 `HAS_PROCESS_STEP`、`AS_FUNCTION`、`EFFECT_OF`、`CAUSE_OF`。同时截图显示边标签压在线上、节点颜色和形状观感较粗糙、画布层级间距不足，影响中文用户理解图谱关系。

本次目标是做轻量修复：中文化图谱展示，并改善现有图谱视觉可读性；不更换图谱库，不重做图谱工作台。

## 目标

1. 图谱画布、图例、详情抽屉中的节点类型和边关系使用中文展示。
2. 边标签清晰可读，不再直接贴在线上造成重叠。
3. 节点样式更统一，保留不同 FMEA 语义类型的颜色区分。
4. 保持现有缩放、适配视图、布局切换、节点详情功能不变。
5. 不改变后端 graph 数据结构和节点/边原始枚举。

## 非目标

- 不新增筛选面板、右侧工作台或复杂交互。
- 不更换图谱渲染库。
- 不调整 FMEA 编辑器的数据结构。
- 不改变枚举入库格式。
- 不处理跨 FMEA 知识图谱总览页的产品功能改版，只修图谱可视化展示层。

## 中文化规则

展示层新增节点类型和边关系翻译函数。函数接收原始枚举，返回中文文案；未知枚举回退为原值，便于发现异常数据。

### 边关系文案

| 原关系 | 中文显示 |
|---|---|
| `HAS_PROCESS_STEP` | 包含工序 |
| `WORK_IN` | 包含作业要素 |
| `HAS_WORK_ELEMENT` | 包含作业要素 |
| `AS_FUNCTION` | 定义功能 |
| `FUNCTION_MAPPED_TO` | 定义功能 |
| `HAS_FAILURE_MODE` | 导致失效 |
| `EFFECT_OF` | 产生影响 |
| `CAUSE_OF` | 由原因引起 |
| `PREVENTED_BY` | 预防控制 |
| `DETECTED_BY` | 探测控制 |
| `OPTIMIZED_BY` | 优化措施 |

### 节点类型文案

| 原类型 | 中文显示 |
|---|---|
| `System` | 系统 |
| `Subsystem` | 子系统 |
| `Component` | 零部件 |
| `ProcessItem` | 结构 |
| `ProcessStep` | 工序 |
| `ProcessWorkElement` | 作业要素 |
| `Function` | 功能 |
| `FailureMode` | 失效模式 |
| `FailureEffect` | 失效影响 |
| `FailureCause` | 失效原因 |
| `PreventionControl` | 预防控制 |
| `DetectionControl` | 探测控制 |
| `RecommendedAction` | 优化措施 |

## 视觉设计

### 节点

- 统一使用圆角矩形作为默认节点形态，避免截图中多种形状混用造成噪音。
- 语义颜色保持区分：
  - 结构/系统/工序：蓝色系。
  - 作业要素/零部件：青色系。
  - 功能：绿色系。
  - 失效模式：红色强调。
  - 失效影响/失效原因：橙黄色系。
  - 控制/措施：绿紫灰色系。
- 节点增加细边框、轻阴影、适度内边距。
- 节点文字居中，过长名称使用省略号，不改变原始数据。

### 边

- 边线使用低饱和蓝灰色，保留箭头方向。
- 边标签使用中文短语，字号控制在 11-12px。
- 边标签增加白色半透明背景、圆角和内边距，减少与边线重叠。
- 标签位置沿用现有布局能力，不引入新的交互。

### 画布和图例

- 画布背景使用浅灰色，减少纯白大面积留白感。
- 图例继续以小卡片方式呈现，但节点类型显示中文。
- 布局间距适度加大，优先减少边标签和节点的重叠。

## 实现方案

1. 在前端图谱展示层增加轻量映射工具，例如 `frontend/src/utils/graphPresentation.ts`。
2. 将节点类型、边关系、节点样式配置集中到该工具中，供图谱页面、图例、详情抽屉复用。
3. 在图谱渲染数据转换处，把边 label 从原始枚举转换为中文展示文案。
4. 在图例和详情抽屉中使用同一节点类型翻译，避免同一类型在不同位置显示不一致。
5. 调整现有图谱节点/边/画布样式，保持功能行为不变。
6. 补充单元测试覆盖节点类型和边关系映射；如果现有测试不足，则至少覆盖工具函数。

## 验证标准

- 前端构建通过。
- 图谱相关映射测试通过。
- 手动打开 FMEA 图谱页，确认：
  - 边关系显示为中文业务短语。
  - 图例和详情抽屉节点类型显示中文。
  - 边标签可读，不明显压在线上。
  - 节点样式更统一，颜色仍能区分 FMEA 语义类型。
  - 缩放、适配视图、布局切换、节点点击详情仍可用。

## 风险和约束

- 如果现有图谱库对边标签样式支持有限，只做其能力范围内的样式优化，不引入大规模重写。
- 对未知枚举回退原值，避免隐藏后端数据异常。
- 本次只处理展示层，不能修复复杂图在极端数据量下天然拥挤的问题。
